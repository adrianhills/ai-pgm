How skills chain together

The leverage isn't any single skill — it's the chains, where one skill's output is the next one's input.

Weekly operating rhythm

tpm-rockstar → comms → status-update

The brief surfaces what shifted and drafts replies; triage prioritizes the inbox sweep; status-update drafts the executive summary.

Risk escalation path

tpm-rockstar → raid → premortem → escalate → decision

The brief detects the signal, RAID captures it, premortem stress-tests failure modes, escalation flags leadership with options, decision documents the call.

New program launch

kickoff → stakeholder-map → workback → kickoff → comms

Brief defines scope and DACE; the map identifies who's in the room; workback computes the schedule from the launch date; comms drafts the announcement.

Meeting to action

zoom-meeting-summary → raid → comms

The transcript becomes a structured summary with action items; the RAID log absorbs the new items with owners; comms posts to the right channel.

Quarterly planning

business-case → roadmap-compare → capacity-plan → kickoff → decision

Business case validates each bet; roadmap compare detects collisions; capacity plan surfaces SPOFs; decision documents the portfolio call.

Incident to learning

rca → retro → premortem → raid

RCA structures the timeline and corrective actions; retro captures reflections; premortem prevents recurrence; RAID absorbs the actions with target dates.

Trust

Built for reliability

A kit you delegate real work to needs reliability primitives and an eval harness — not just good prompts.

🔍

Reliability primitives

Verify-coverage checks, freshness gates on stale data, and provenance back to Slack / Smartsheet / Drive / Jira / Calendar. Every claim traces to a source.

🧪

Eval harness

Every skill is A/B tested against the same request with no skill loaded. If it isn't materially better than the base model, it doesn't ship.

📈

Competency mapping

Each skill maps to the TPM/PGM competency it builds, so delegated work also reinforces the competencies you're promoted on.

🧠

Program memory

A per-program brief re-injected at run start, plus an append-only decisions ledger with mandatory evidence refs. Contradictions are surfaced, never silently resolved.
